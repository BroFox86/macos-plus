Введение

О руководстве

Данное руководство наполовину состоит из переведенных мною инструкций c английского языка и предполагает установку на ПК официального образа macOS Sierra с магазина AppStore при использовании программ сайта tonymackx86.com.

Оригинальными источниками послужили:

  • UniBeast: Install macOS Sierra on Any Supported Intel-based PC
  • How to Install macOS Sierra Final on VMware on Windows

Помимо текстового перевода, я добавил свои инструкции по подготовке компьютера и решению возможных проблем, с которыми может столкнуться любой начинающий "хакинтошник". То есть информацию, которая как правило не приводятся в других статьях на эту тему, и которую новичкам приходится долго искать самостоятельно. В руководство также добавлены мои советы, примечания и ссылки на дополнительную информацию.

О сложностях хакинтоша

С помощью моего руководства получится установить macOS Sierra на большинство современных ПК на базе процессоров intel. Однако сама по себе настройка хакинтоша сопряжена со множеством проблем, решение всех из которых это руководство охватить не в состоянии: macOS расчитана на фирменное оборудование Apple, тогда как у пользователей компьютеров, настраивающих macOS на ПК, конфигурация систем разнится между собой. Поэтому проблемы возникают у всех разные. Исходя из этого, искать решения возникшей проблемы придется в сети, желательно на зарубежных специализированных форумах, таких как упомянутый tonymackx86. Либо пытать счастья на applelife.ru, если уж совсем не "дружите" с английским. Весьма вероятно, что человек с аналогичной проблемой уже обращался за помощью и решил ее определенным образом.

Шаг 1: Перед тем, как начать

Подготовка

  • Имейте наготове USB-накопитель объемом от 8Gb
  • Зарегистрируйтесь на www.tonymacx86.com
  • Загрузите последние версии UniBeast и MultiBeast с раздела Downloads
  • Сделайте резервную копию всех важных данных

Касательно параметров системы:

  • Поддерживает ли система (материнская плата) UEFI, либо использует BIOS?
  • Поколение процессора intel Core. Это может быть Sandy Bridge, Ivy Bridge, Haswell или Skylake. Возможна работа также и чуть более старых процессоров
  • Аудиокодек материнской платы, если используете встроенное аудио. На системах с чипсетом intel должен быть один из кодеков Realtek ALC

MacOS рекомендуется устанавливать на отдельный жесткий диск, на котором не установлена основная ОС, во избежание ошибок пользователя, в результате которых может быть стерт загрузочный раздел основной ОС или произойти другие неприятности. Также это поможет избежать возможных ошибок при установке macOS. Жесткий диск при этом будет полностью отформатирован.

Далее я также разберу способ установки macOS Sierra на отдельно подготовленный раздел неосновного жесткого диска, однако такой способ займет больше времени и может быть проблемным.

Сбор информации о системе

  1. Скачайте условно-бесплатную диагностическую утилиту AIDA64
  2. В разделе Суммарная информация узнайте чипсет своей материнской платы
  3. И тип "BIOS" - AMI UEFI или AWARD BIOS

    Примечание: В краткой форме основные различия UEFI от BIOS описаны в статье на сайте vindavoz.ru

  4. На той же вкладке указан аудиокодек материнской платы
  5. Название сетевой карты в разделе Сеть > Сеть PCI/PnP

Шаг 2: Настройка VMWare

Перед тем, как начать

Для того, чтобы установить официальный образ macOS Sierra на ПК, потребуется для начала загрузить его через цифровой магазин AppStore. Это можно сделать на ОС Windows с помощью "виртуальных машин" - программ, которые эмулируют работу одной ОС внутри другой. В данном руководстве будет рассматриваться бесплатная версия VMWare Workstation Player, возможностей которой вполне хватит для описанных в данном руководстве действий. Запущенная на виртуальной машине macOS отображается как обычная программа для Windows - в окне или в режиме полного экрана.

Совет: Загрузка официального образа macOS с AppStore возможна также и на любом компьютере Apple Mac, работающем на OS X 10.7.5 или более поздней версии. При наличии такой возможности можно пропустить главу настройки виртуальной машины.

Примечание: macOS Sierra в качестве виртуальной системы не предложит пользователю все свои функции и возможности по техническим причинам, в следствие чего не способна оставить полноценные впечатления от своей работы. Тем не меннее в таком режиме можно бегло освоиться с интерфейсом и принципами незнакомой ОС.

Для установки macOS Sierra в качестве виртуальной ОС понадобиться скачать следующие файлы:

  • Программу VMWare Workstation Player: www.vmware.com/products
  • Образ macOS Sierra для виртуальной машины: Google Drive / Яндекс Диск
  • Патчи Patch Tool и VM Tool: https://goo.gl/2cR8gm

Установка и настройка

  1. Установите Workstation Player
  2. Распакуйте архив с помощью Winrar или 7zip
  3. Запустите win-install.cmd от имени администратора и пропатчите VMWare Workstation Player
  4. Откройте Workstation Player и нажмите Create new virtual machine, затем выберите Guest OS: Apple Mac OS X и Version: Mac OS X 10.11 или 10.12
  5. Выберите Edit Virtual Machine Setting, затем Hard Disk (SATA) 40 GB и удалите виртуальный диск нажав на Remove
  6. Следуйте цепочке действий: Add > Hard Disk > SATA, после этого выберите Use an Existing Virtual Disk
  7. Найдите и откройте загруженный образ macOS Sierra ( ...\macOS Sierra.vmdk )
  8. Рекомендую выделить 4Гб оперативной памяти для нормальной работы macOS Sierra на виртуальной машине. Если у Вас 4-ядерный процессор, выделите ей 2 ядра
  9. Откройте *.VMX файл с помощью блокнота
  10. Скопируйте и вставьте последней строкою фрагмент кода: smc.version ="0"
  11. Запустите Workstation Player и установите macOS Sierra
  12. Смонтируйте образ VM Tool.iso, следуя цепочке действий: Workstation Player > Removable Devices > CD/DVD > Settings и установите… Либо загрузите образ WMware Tool прямо из браузера Safari
  13. Готово! macOS Sierra работает в среде VMware на Windows :)

    Совет: Не удаляйте настроенную Workstation Player по крайней мере до того момента, пока не установите macOS в полноценном виде на ПК - виртуальная машина может еще пригодиться!

Шаг 3: Создание загрузочного USB

  1. Откройте Mac AppStore
  2. Найдите и загрузите образ macOS Sierra

Приложение установки Install macOS Sierra появится в списке доступных программ.

Сделайте глубокий вдох. Это руководство довольно простое, но из-за спешки можно упустить важные детали!

  1. Вставьте USB-накопитель
  2. Откройте Finder/Программы/Утилиты/Дисковая утилита
  3. Выберите накопитель в левой колонке
  4. Нажмите кнопку Стереть. Все данные этого USB будут стерты, а сам диск отформатирован
  5. Введите любое имя на латинице, например, USB (имя можно изменить позже)
  6. Выберите формат: OS X Extended (Журналируемый)
  7. Выберите схему: GUID Partition Map (GPT)
  8. Нажмите Стереть, затем Готово
  9. Установите последнюю версию UniBeast и запустите
  10. Нажимайте Continue, Continue, Continue, Agree, пока не дойдете до Destination Select
  11. На этапе Destination Select выберите USB-накопитель и нажмите Continue

    Совет: У UniBeast проблемы с обнаружением накопителей большой емкости, таких как внешние жесткие диски. Для того, чтобы обойти это ограничение, я создал 16Гб раздел с файловой системой HFS с помощью Paragon Hard Disk Manager, а остальное пространство оставил неразмеченным.

  12. На экране Select OS Installation выберите Sierra и нажмите Continue
  13. На экране Bootloader Options выберите UEFI Boot Mode или Legacy Boot Mode. UEFI Boot Mode рекомендуется для 7/8/9/100 серии чипсетов Intel и для всех поддерживающих UEFI материнских плат. Legacy Boot Mode рекомендуется для 5/6 серий чипсетов и всех систем на основе BIOS

    Примечание: Чипсеты Intel для материнских плат 100-й серии - это такие модели, как h370, B150 и т.д. Чипсеты 9-й серии - H97, Z97...

  14. (Опционально) На экране Graphics Configuration список опций для старых видеокарт и встроенных решений. Ознакомьтесь с их описанием
  15. Проверьте параметры установки, затем нажмите Continue и Install.

UniBeast создаст загрузочный USB, используя лучший на данный момент загрузчик - Clover. Процесс может занять до 10 минут в зависимости от системы и скорости накопителя. Постарайтесь не прерывать этот процесс. Также не рекомендую выходить из виртуальной машины, в случае, если установка происходит через нее.

Примечание: Для того, чтобы macOS запустилась на обычном ПК, ей нужен сторонний загрузчик. То есть загрузчик необходим, чтобы загрузить macOS на не-Apple оборудовании. Clover является проектом русскоязычного разработчика Slice и в данное время является единственным полноценно работающем решением для "хакинтоша" во всем мире.

  1. Переместите архив с MultiBeast на созданный загрузочный накопитель.

Совет: При желании вернуть USB-накопитель в прежнее состояние, необходимо будет не только полностью его отформатировать в NTFS/FAT32, но и, возможно, изменить его схему на предыдущую - MBR. Если не будет изменен тип схемы, то "флешку" могут не увидеть некоторые устройства - DVD-проигрыватели, старые компьютеры и т.д. Такие операции успешно выполняет бесплатная программа MiniTool Partition Wizard Free или условно-бесплатная Paragon Hard Disk Manager.

Шаг 4: Подготовка компьютера

Предупреждение: Автор не несет ответственности за ваши утерянные данные, расстроенное оборудование и кривые руки. Данное руководство - это по большей части перевод зарубежных "гайдов" с моими дополнениями, которые пригодились лично мне при настройке "хакинтоша" на своей системе.

Рекомендуемые настройки для BIOS/UEFI

Если Вы настраиваете macOS на системе с UEFI, то настройка материнской платы не будет сложной. Для других систем понадобится сбросить настройки BIOS на стандартные (Optimized Defaults) и изменить режим работы жесткого диска на AHCI. Ниже представлены общие рекомендации для всех материнских плат. За ними следуют рекомендации по настройке для материнских плат от разных производителей, которыми поделились участники форума tonymackx86.com.

Для доступа в BIOS/UEFI нажмите и удерживайте определенную клавишу во время загрузки системы. Как правило это Delete, F9 или F11. Вам придется пройтись по разделам и применить указанные ниже рекомендации.

  1. Для начала сбросьте настройки BIOS на стандартные (Load Optimized Defaults)
  2. Выключите VT-d?
  3. Выключите CFG-Lock
  4. Выключите Secure Boot Mode?
  5. Измените опцию OS Type на Other OS
  6. Выключите IO SerialPort?
  7. Включите XHCI Handoff?
  8. Если в системе набор логики (чипсет) Intel 6-й серии или Intel x58 с AWARD BIOS, выключите порты USB 3.0
  9. Сохраните настройки и перезагрузитесь (Save Changes and Reboot).

Настройки для Gigabyte

Рекомендации для UEFI:

  1. Нажмите и удерживайте клавишу (Delete) во время загрузки системы
  2. Сбросьте настройки UEFI на стандартные (Load Optimized Defaults)
  3. (Опционально) Измените X.M.P. Memory Profile на Profile1
  4. Выключите VT-d?
  5. Сохраните настройки и перезагрузите компьютер (Save Changes and Reboot).

Рекомендации для BIOS:

  • Используйте только 1 видеокарту в 1 PCIe разъеме с 1 подключенным монитором
  • Отключите любые жесткие диски за исключением пустого, на который будет установлена macOS
  • Отключите любую USB периферию, кроме клавиатуры и мыши
  • Отключите любые PCI-карты за исключением видеокарты - они могут быть несовместимы с macOS
  • Рекомендуется использовать пустой жесткий диск
  1. Сбросьте настройки BIOS на стандартные (Load Optimized Defaults)
  2. Измените режим работы жесткого диска на AHCI
  3. Измените приоритет загрузки: поставьте загрузку с CD-ROM на первое место
  4. (Опционально) Измените X.M.P. Memory Profile на Profile1
  5. Измените режим HPET? на 64-bit mode.

Настройки для ASUS на UEFI

Для ASUS Z97-A рекомендуется версия прошивки v1204 или новее. В настройках проверить, отключена ли CFG lock.

Настройки для MSI на UEFI

  1. Сбросьте настройки UEFI на стандартные (Load Optimized Defaults)
  2. Выключите CFG-Lock
  3. Выключите VT-d?
  4. Сохраните настройки и перезагрузитесь (Save Changes and Reboot).

В более наглядном виде это показано на tonymacx86.com.

Подготовка жесткого диска (только для UEFI)

Совет: При готовности выделить отдельный жесткий диск для полного форматирования в файловую систему HFS+, с удалением всех его данных, пропустите главу подготовки жесткого диска.

Подготовка жесткого диска будет проходить в Windows. Для установки macOS Sierra на отдельный раздел жесткого диска потребуется система с UEFI загрузкой и жесткий диск (HDD/SSD), размеченный в GPT. Если схемой диска является MBR, ее необходимо изменить, - что невозможно без использования специальных программ, таких как Paragon Hard Disk Manager. Тем не менее не стоит полностью доверять изменение схемы разделов стороннему ПО и сделать резервную копию важных данных. В зависимости от количества данных, этот процесс может занять довольно длительное время. macOS Sierra теоретически можно установить и на раздел диска, размеченного в MBR на систему с UEFI или, возможно, BIOS загрузкой, однако мое руководство не рассматривает такие варианты.

Примечание: Подробнее о схеме разделов MBR и пришедшей ей на смену GPT изложено в статье на сайте windowsten.ru.

Предупреждение: Создайте резервную копию всех важных данных - документов, фото, "сохранок" игр и иных файлов, которых невозможно будет восстановить или загрузить откуда-либо в случае утери. Будьте предельно осторожны при работе с разделами: по невнимательности можно удалить разделы с важными данными и операционными системами!

Для установки macOS на отдельный раздел жесткого диска в системе c UEFI загрузкой понадобиться создать 2 раздела: загрузочный (EFI) и основной.

  1. Для создания и форматирования разделов понадобится специальная утилита. Я рекомендую мощное решение: условно-бесплатную программу Paragon Hard Disk Manager. Загрузите ее с официального сайта и установите
  2. (Опционально) Измените схему на GPT, если это необходимо
  3. Измените размер раздела целевого диска. После изменения размера должна появиться незанятая область до или после раздела. Размер незанятого пространства равняется месту, выделяемому для macOS . Рекомендую выделить для macOS Sierra как минимум 30Гб
  4. Создайте в неразмеченной области раздел FAT32 размером 200Мб, назовите его EFI и не назначайте ему букву
  5. Оставшееся пространство разметьте в Apple HFS, назовите его любым именем на латинице (я назвал его macOS) и не назначайте ему букву
  6. После того, как все шаги пройдены, внимательно проверьте свои действия и нажмите Применить - это запустит процесс применения всех указанных выше действий.

Шаг 5: Установка macOS Sierra на ПК

Вы почти закончили! Все, что осталось для установки - загрузиться с USB-накопителя и установить macOS! Для лучших результатов вставьте накопитель в разъем USB 2.0.

  1. Включите компьютер
  2. Загрузитесь с USB, выбрав соответствующий пункт в меню загрузки UEFI. Или предварительно выставите для этого загрузку с USB/CD-ROM в BIOS
  3. Должен появиться интерфейс загрузчика Clover. Запустите установку macOS Sierra с USB-накопителя. В моем случае - Boot Mac OS X from USB
  4. Когда увидете экран установки macOS, выберите необходимый язык системы
  5. (Опционально) Для первичной установки macOS на ПК необходимо отформатировать целевой диск или пропустить данный этап, если заранее был подготовлен раздел для установки macOS в Подготовка жесткого диска.
    1. В верхнем меню выберите Утилиты и откройте Дисковую утилиту
    2. Выберите целевой жесткий диск в левой колонке
    3. Нажмите кнопку Стереть
    4. Введите имя на латинице, например, macOS (имя можно изменить позже)
    5. Формат: OS X Extended (Журналируемый)
    6. Схему разделов: GUID Partition Map (GPT)
    7. Нажмите Стереть
    8. Выйдите из Дисковой утилиты.
  6. Когда установщик попросит выбрать место установки, выберите созданный раздел macOS
  7. После завершения система будет автоматически перезагружена
  8. Загрузитесь с USB-накопителя
  9. На экране загрузчика Clover выберите раздел с установленной macOS Sierra. В моем случае - Boot MacOS from macOS
  10. Завершите установку macOS Sierra

Примечание: В начале, при первичной установке macOS на ПК, возможно, не заработает большая часть системного оборудования. Это касается звука, некоторых USB портов и сетевых устройств. Кроме того, видеокарта будет работать в стандартном режиме - что означает медленную работу интерфейса и отсутствие некоторых визуальных эффектов.

Шаг 6: Пост-установка с MultiBeast

Проверка загрузочного раздела (только для UEFI)

Рекомендую проверить созданный Вами (см. Подготовка жесткого диска) или установщиком (см. Установка macOS Sierra) EFI раздел. Для этого необходимо его смонтировать, поскольку разделы EFI по умолчанию скрыты для всех ОС.

  1. Загрузите EFI Mounter и установите ее на macOS, распаковав утилиту и переместив ее в /Программы
  2. На сообщение системы безопасности отвечаем Открыть

    Примечание: Для того, чтобы в macOS разрешить установку программ, загруженных со стороны - т.е. не с AppStore - необходимо отметить соответствующий пункт раздела "Защита и безопасность" в "Системных настройках".

  3. EFI Mounter обнаружит разделы EFI на всех жестких дисках и подключенных носителях. В моем случае, отображены 2 системных диска - SSD (твердотелый накопитель) и HDD (жесткий диск). SSD-диск указан как Disk0s1, где s1 - это первый раздел данного диска, который и является EFI - загрузочным разделом в UEFI. Выбираем нужный раздел и нажимаем Mount
  4. Подтверждаем выбор. В "проводнике" Finder в левой колонке к списку монтированных накопителей добавился EFI. Это и есть созданный Вами/системой загрузочный раздел!

На приложенном скриншоте в разделе уже имеется директория EFI с загрузчиком Clover внутри нее. Но при первичной установке macOS на ПК этот раздел должен быть пустым. Если на разделе есть какие-либо файлы, то это могут быть данные для загрузки Windows, которых трогать не стоит! Вам может смонтироваться не тот загрузочный раздел EFI в следующих случаях:

  • Вы установили macOS на один жесткий диск с ОС Windows и по ошибке смонтировали EFI с файлами этой ОС
  • На целевом жестком диске по какой-либо причине вообще нет раздела EFI и смонтировался аналогичный раздел для Windows из другого физического диска
  • По ошибке подключился EFI-раздел с загрузочного USB

Предупреждение: Не стоит оставлять подключенными разделы EFI, не имеющие отношения к macOS. Отключайте подобные разделы, добавленные по ошибке: они могут быть переписаны некоторыми программами по настройке хакинтоша.

Настройка оборудования с помощью MultiBeast

MultiBeast - пакет программ "все в одном", который разработан упростить настройку macOS на ПК. В состав программы входит загрузчик Clover, кексты? для настройки оборудования и скрипты для настройки загрузчика. MultiBeast настроит загрузку macOS Sierra с жесткого диска, заместо того, чтобы всякий раз делать это с загрузочного USB. Кроме этого, MultiBeast поможет настроить встроенное аудио, сетевую карту, видеокарту и другое внутреннее оборудование компьютера. То есть основное отличие MultiBeast от UniBeast состоит в том, что UniBeast создает загрузочный накопитель для первичной установки macOS, тогда как MultiBeast создает загрузчик на жестком диске и настраивает как его, так и macOS под внутренее оборудование компьютера.

Каким образом работает MultiBeast:

  • В случае с UEFI-загрузкой, автоматически монтирует раздел EFI жесткого диска
  • По аналогии с созданном UniBeast загрузочном USB, MultiBeast создает директорию с загрузчиком Clover, но на EFI разделе жесткого диска
  • Устанавливает необходимые кексты? в системную папку macOS или папку загрузчика
  • Добавляет необходимые параметры в виде строк кода в файл конфигурации загрузчика Clover - config.plist

Совет: Выбирайте опции MultiBeast с уверенностью, что они соответствуют Вашей системе. В случае сомнений не устанавливайте их.

Примечание: MultiBeast предназначен для настройки под macOS только внутренних устройств. Для многих внешних устройств USB, таких как внешние звуковые карты, клавиатуры/мыши и адаптеры Wi-Fi - искать драйвера (кексты) для macOS в первую очередь надо на официальных сайтах производителей. Если в списках MultiBeast Вы не увидите кекстов для своего внутреннего оборудования, придется искать его в сети, на форуме tonymacx86.com. Таким образом я нашел кекст для своей сетевой карты Atheros E2400.

  1. Запустите MultiBeast
  2. Выберите Quick Start. UEFI Boot Mode рекомендуем для систем на базе 7/8/9/100 серии чипсетов Intel и всех систем с UEFI загрузкой. Legacy Boot Mode рекомендуем для 5/6 серий и всех систем на базе BIOS
  3. На вкладке Audio представлен список опций для аудиокодеков Realtek ALC. Материнские платы на чипсетах Intel как правило используют эти кодеки. Также представлены опции для звука, реализуемого посредством HDMI. Выберите подходящий вариант для своей системы, отметив только один пункт. Если в системе чипсет Intel 100-серии, то дополнительно отметьте и соответствующую опцию. Если так и не смогли определить свой аудио-кодек, то вероятно подойдет один из вариантов в группе Universal > VoodooHDA
  4. На вкладке Disc представлены опции для сторонних (3rd Party) SATA-контроллеров жестких дисков, а также Intel Generic AHCI SATA, которые подходят только для определенных чипсетов (см. описание)
  5. В Misc представлен важный для "хакинтошей" кекст FakeSMC, эмулирующий фирменную микросхему SMC?, установленную на материнских платах компьютеров Apple. Одна из подходящих Вашей системе опций должна уже быть отмеченной по умолчанию.
    • Кексты VoodooTSCSync предназначены только для процессоров c разъемом LGA2011
    • FakeSMC HWMonitor (прим.: возможная несовместимость с видеокартами Nvidia Pascal!) предназначен для получения данных о температуре оборудования и скоростях вращения систем охлаждения.
  6. На вкладке Network выберите подходящую сетевую карту
  7. На USB представлен список опций для портов USB:
    • 3rd party USB 3.0 - для сторонних контроллеров USB 3.0
    • 7/8/9 Series USB Support - для USB 2.0/3.0 для чипсетов Intel 7/8/9 серии
    • Increase Max Port Limit - для USB 2.0/3.0 для чипсетов Intel 9 и 100-й серии
  8. Переходим на вкладку Bootloaders и видим 2 пункта. Одна из подходящих опций уже должна быть отмеченной в соответствии с выбором на Quick Start
  9. Вкладка Customize разбита на 3 раздела. Первый раздел - Graphics Configuration
    • Inject NVIDIA - опция для старых видеокарт NVIDIA GeForce серии 8xxx, 9xxx, 2xx, 4xx, 610 и 630
    • Inject ATI - для старых видеокарт ATI, таких как 4ххх серия
  10. Раздел SSDT конфигурации содержит только опции для процессоров семейства Sandy Bridge
  11. Раздел System Definitions содержит ряд опций, которые маскируют Ваш ПК под настоящий компьютер Mac, тем самым открывая доступ к почти всем сервисам Apple - iCloud и AppStore - а также нужны для нормальной работы macOS в целом. К счастью, Вам не придется выбирать подходящий системе вариант, так как он уже должен быть отмечен - раскройте списки и убедитесь в этом сами

    Совет: MultiBeast не способен корректно настроить хакинтош для работы сервисов Apple. Для нормальной работы iCloud и AppStore необходима самостоятельная настройка загрузчика Clover через специальные утилиты. Не рекомендуется заходить в эти сервисы, пока не была произведена такая настройка: может быть заблокирована Ваша учетная запись Apple ID. Что касается iMessage и FaceTime, то для их завода потребуются определенные номера идентификации (не серийные) с настоящего компьютера Mac.

  12. Нажмите Print или Save для создания резервной копий конфигурации, так как MultiBeast является установщиком, но не деинсталятором
  13. Нажмите Build, затем Install.

Установка драйверов видеокарты

Если у Вас видеокарта GeForce GTX 7xx, 9xx, TITAN X или выше, то macOS Sierra не поддерживает их по умолчанию. Вам потребуются альтернативные драйвера Alternate NVIDIA drivers и утилита Clover Configurator для того, чтобы "прописать" эти драйвера в загрузчике Clover.

  1. Загрузите последнюю версию Alternate NVIDIA Graphics Drivers для своей версии macOS и установите ( прим.: по ссылке версии для 10.12.3). Пока не перезагружайтесь
  2. Загрузите и установите Clover Configurator
  3. Найдите папку загрузчика Clover, созданную MultiBeast на жестком диске. Она должна располагаться на разделе EFI. Все подключенные разделы можно увидеть в левой колонке Finder (см. Проверка загрузочного раздела)
  4. Откройте config.plist через Clover Configurator
  5. В Clover Configurator перейдите на вкладку System Parameters и отметьте опцию NvidiaWeb
  6. Сохраните изменения и закройте утилиту.

По окончании процедуры безопасно извлеките загрузочный USB и перезагрузите компьютер. Теперь Вы загрузитесь в macOS без USB. В вариантах загрузки UEFI должен появится пункт для загрузки с жесткого диска, на котором установлена macOS Sierra. На экране Clover выберите Boot MacOS from macOS.

В случае с Legacy загрузкой (BIOS), система должна быть настроена на загрузку с того жесткого диска, на котором установлена macOS.

Вжуух! И у Вас установлена и более-менее настроена (надеюсь) macOS Sierra!

Не стоит беспокоиться за то, что после перезагрузки что-либо не заработает: сложности при настройке хакинтоша - вполне закономерное явление...

О заводе сервисов AppStore и iCloud, а также о других сложностях, которые могут возникнуть, в частности проблемах со встроенным звуком, я расскажу в следующих обновлениях своего руководства.

Решение проблем

Unibeast: Volumes are Mounted

На конечном этапе создания загрузочного USB через UniBeast может появиться сообщение с заголовком "Volumes are Mounted". Это преграда, которая сделает невозможным дальнейшее продвижение по инструкции.

Решение:

  1. Войдите в Finder, в строке меню выберите переход > переход к папке
  2. Введите во всплывающем окне /Volumes и найдите в открывшемся списке раздел, имеющий отношение к macOS
  3. Переименуйте его на что нибудь другое, например на Other OS X Base System
  4. Закройте UniBeast и повторите всю процедуру создания загрузочного USB заново

Паника ядра

Если установка macOS на ПК прервется экраном с перечеркнутым кружком - это паника ядра - преграда, с которой сталкиваются многие люди, настраивающие хакинтош. Причин может быть много, однако в моем случае данная проблема сразу же возникала при попытке установке macOS Sierra на отдельный раздел жесткого диска (HDD) на системе с UEFI. Если Ваша ситуация во многом аналогична, вероятно, проблему поможет решить нижеследующая инструкция.

  1. Загрузитесь в Windows
  2. Запустите настроенную в первой главе виртуальную машину с macOS
  3. Подключите созданный UniBeast загрузочный USB
  4. Откройте раздел EFI (он примонтируется как отдельный накопитель) и проследуйте по пути: EFI/Clover/driversolutions4uefi
  5. Скачайте альтернативный файл OsxAptioFixDrv-64.efi и скопируйте его в папку driversolutions4uefi загрузчика Clover. Удалите из папки другие файлы, начинающиеся с названия "OsxAptioFix..."
  6. Попытайтесь снова установить macOS Sierra.

Если вышеуказанный способ не помог, надо загрузиться в режиме вывода сообщений – Verbose mode:

  1. Загружаемся в Clover
  2. На загрузочном экране выделяем Boot MacOS from macOS
  3. Нажимаем на Пробел и выбираем Boot Mac OS X in Verbose Mode

Экран должен заполниться информацией об ошибке, а Вам остается его сфотографировать и искать решение в сети.

Темный экран при выходе из сна

При отсутствии изображения после пробуждения компьютера должна помочь следующая процедура:

  1. Запустите Clover Configurator
  2. Найдите папку загрузчика Clover. Она должна располагаться на разделе EFI. Все подключенные разделы можно увидеть в левой колонке Finder (см. Проверка загрузочного раздела)
  3. Откройте config.plist через Clover Configurator
  4. Перейдите на вкладку Boot и измените значение darkwake. Рекомендую выставить darkwake=8
  5. Сохраните изменения и закройте утилиту
  6. Перезагрузите компьютер
